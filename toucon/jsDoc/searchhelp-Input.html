searchhelp-Input

new searchhelp-Input(serviceUrlopt, searchHelpIdopt, keyFieldopt, keyBindingPathopt, valueFieldopt, valueTypeopt, valueBindingPathopt, filterFieldsopt, searchFieldsopt, titleopt, tooManyResultsTitleopt)

The searchhelp.Input is an assisted input field which pops up an SAP F4-style dialog with filter and search options.
Parameters:
Name Type Attributes Description
serviceUrl string <optional>
 the URL of the data service
searchHelpId string <optional>
 name of the backend value help
keyField string <optional>
 name of the SAP column whose data should be written back to the data model
keyBindingPath string <optional>
 path for the hidden key input (where the keyField's value should be written to)
valueField string <optional>
 name of the SAP column from which the display value should be retrieved
valueType object <optional>
 for type validation (only if the KEY is not used and if this is not valueHelpOnly)
valueBindingPath string <optional>
 path for the main input field
filterFields Array.<string> <optional>
 names of the SAP columns which should be displayed in the results dialog
searchFields Array.<string> <optional>
 names of the SAP columns on which an eventual pre-filter search should be made
title string <optional>
 title of the TableSelectDialog which displays the results
tooManyResultsTitle string <optional>
 title of the Dialog which displays filter parameters if too many results have been returned (also used by the TableSelectDialog if the search still returns too many results.

(protected, static) onValueHelpRequest(oEventopt)

Retrieves the title from the backend by calling [serviceUrl]/SearchHelpSet([searchHelpId])?$select=Ddtext&$format=json
Retrieves the available attributes from the backend by calling [serviceUrl]/AttributeSet?$select=Fieldname,ScrtextL,Datatype,Shlpinput,Shlpoutput,Shlpselpos,Shlplispos&$format=json
Finally retrieves the number results by calling [serviceUrl]/ValueSetBySearchString/$count?searchHelpName='[searchHelpId]'&searchString='[_selectOptions] (the select options are compiled by the eventual search which the user will make in order to restrict the number of results
If the number of results is too big (currently hard-coded to >100), then the search/select dialog will be displayed, otherwise the results dialog will be opened.

(protected, static) renderer(oRmopt, oControlopt)

Checks if the valueHelpOnly property is true or false and then binds the KEY as a TwoWay binding to the hidden key field while binding the VALUE as a OneWay binding to the visible input field (this enables us to display a nice value, while writing a technical value back to the model) OR binds the visible input field directly to the model and attaches the type (for form validation).
This function attaches the onValueHelpRequest function to the valueHelpRequest event and then renders the hidden key field and finally calls the sap.m.InputRenderer.render function for rendering the rest of the control.

(protected, static) showResultDialog(forceRefreshopt)

Loads the results table (if it does not already exist) and then opens it.
Results are retrieved by calling[serviceUrl]/ValueSetBySearchString/?searchHelpName='[searchHelpId]'&searchString='[selectOptions]'&$select=Fieldname,Recordpos,Fieldval,Retfield&$format=json

Please note that the ODATA which is initially loaded is treated like a JSON object and transformed into a flattened JSONModel before the column list items are generated.

Please note that the results dialog will use the normal title or the tooManyResultsTitle if the number of results exceeds the hard-coded value of 100.
Parameters:
Name Type Attributes Description
forceRefresh boolean <optional>
 will cause the destruction of columns and items and therefore trigger the reload of the results table.
