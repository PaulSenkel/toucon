Class: toucon-Table

toucon-Table

new toucon-Table(actionModeopt, actionOnSelectopt, editModeopt, selectionModeopt, selectionBehavioropt)

The Table control enables us execute actions on row selection, to make a row editable on row selection and will also highlight columns which have the editable flag set to true.
This table enables us a more interactive experience as we can
a) execute for example an "open" action on a row if the table is not in edit mode
b) make a line editable on row selection after we switched the table to edit mode
c) clearly highlight columns in which we want users to enter data

As ODATA models do not let us modify more than one data set at the same time, this table is in Single selection mode by default and fires an event before a new line is made editable. This enables us to save changes and refresh our model before we move to the next line.
Parameters:
Name Type Attributes Description
actionMode boolean <optional>
 default is "false"; if "true", then the actionOnSelect function will be executed when a row is being selected
actionOnSelect function <optional>
 the function which will be executed on row selection if actionMode is true
editMode boolean <optional>
 default is "false"; if "true" all controls in the row will be made editable or enabled. Controls of all other rows will be inactivated.
selectionMode sap.ui.table.SelectionMode <optional>
 defaulting to sap.ui.table.SelectionMode.Single
selectionBehavior sap.ui.table.SelectionBehavior <optional>
 defaulting to sap.ui.table.SelectionBehavior.RowSelector -> one must click on a row header in order to select the row

(protected, static) editRow(selIdopt, silentopt)

Fires the beforeMakingRowEditable event,
then checks if there is a previously selected row (which we are now leaving) that has error value states on its controls and fires the onValueStateError event if this is the case,
if no errors have been found then this function runs over all rows and attempts to set the controls' editable flag to false (any row) or true (selected row).
On the selected row it will also apply previously saved valueStates so as to highlight the controls which caused an error (this makes sure that we see highlighted error states after a rerender).
If controls cannot be made editable, then as a fallback it does the same for the enabled flag.
At the end the current row is memorized as currentEditableRow and the memorized valueStates are reset to empty.

(protected, static) onAfterRendering()

Runs over all columns and rows in order to highlight column headers and cells by attaching the touconTableEditableColumn/Cell CSS classes if these columns are flagged as editable.
Also attaches the touconTableSelectToOpen/Edit classes according to the current action/edit mode and attaches the editOnSelect function or the actionOnSelect function to the row selection event.
Also makes the currently remembered editable row editable again by calling the editRow function in silent mode (no events will be fired for this editRow call)
